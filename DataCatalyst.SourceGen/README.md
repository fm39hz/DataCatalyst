# DataCatalyst.SourceGen

A compile-time Roslyn Incremental Generator for DataCatalyst.

## ⚡ How it Works

The generator scans the current compilation and referenced assemblies for:
1. Structs decorated with the `[DataComponent]` attribute.
2. Classes implementing the `IDataPlugin` interface (with optional `[DataPlugin]` attributes).

It emits a `PrimitiveRegistrations.g.cs` containing `[ModuleInitializer]` methods, automatically registering these types into `PrimitiveRegistry` and `PluginRegistry` before your application's entry point runs.

This guarantees:
* **Zero Runtime Reflection**: Crucial for Native AOT compilation and trimming.
* **Topological Sorting**: Discovered plugins are sorted topologically based on their `DependsOn` dependency declarations, ensuring correct initialization order.

## 📄 Example

### Input C#
```csharp
namespace MyGame;

[DataComponent]
public struct Health
{
    public float Max;
}

[DataPlugin(DependsOn = [typeof(OtherPlugin)])]
public class MyPlugin : IDataPlugin { }
```

### Generated Code
```csharp
// <auto-generated />
using System.Runtime.CompilerServices;

namespace DataCatalyst.Generated;

internal static class PrimitiveRegistrations
{
    [ModuleInitializer]
    internal static void RegisterPrimitives()
    {
        global::DataCatalyst.Core.PrimitiveRegistry.Register<global::MyGame.Health>();
    }

    [ModuleInitializer]
    internal static void RegisterPlugins()
    {
        // Ordered topologically (OtherPlugin is registered before MyPlugin)
        global::DataCatalyst.Core.PluginRegistry.Register<global::MyGame.OtherPlugin>();
        global::DataCatalyst.Core.PluginRegistry.Register<global::MyGame.MyPlugin>();
    }
}
```
