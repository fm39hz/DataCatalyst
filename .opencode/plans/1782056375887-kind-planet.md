# Redesign Plan: Schema-Driven DataCatalyst

## Core Insight

**Serialization is not the hard part.** The hard part is knowing what types to use. Schema solves this.

```
Loader (JSON/CSV/MessagePack) 
  → reads raw data
  → looks up each field in SchemaBuilder (type tells parser what to do)
  → creates DataEntry with typed fields
  → done. No hardcoded field names anywhere.
```

## Schema Registration (by Core + Plugins)

```csharp
// Core registers built-in fields at environment creation
schema.Register<string[]>("inherits");   // type = string[] → loader parses as array
schema.Register<int>("layer");            // type = int → loader parses as number

// Plugin registers its fields at init
schema.Register<string>("Concept");       // type = string → loader parses as string
```

**Loader doesn't know about "inherits", "Concept", or "layer".**
It only knows: "field named X has type Y — parse accordingly."

## DataEntry — frozen after construction

```csharp
public sealed class DataEntry {
    public string Key { get; }
    public IReadOnlyDictionary<Type, object> Components { get; }
    public IReadOnlyDictionary<Type, object> Fields { get; }
    
    public T GetField<T>() where T : notnull;        // throws if missing — schema guaranteed
    public T GetField<T>(T fallback) where T : notnull;
    public bool TryGetField<T>(out T value) where T : notnull;
}
```

- No `SetField<T>()`. No `MutableComponents`. 
- All fields set at construction via builder, then frozen.
- `typeof(T)` = key. Each field has unique type (`string[]`, `string`, `int`, etc.)

## Loader Contract

```csharp
public interface IDataLoader {
    LoadResult LoadDirectory(string path, DataCatalystEnvironment env);
}

// Loader receives env with SchemaBuilder
// For each field in raw data:
//   1. Look up field name in schema → type
//   2. Parse raw value to that type
//   3. Store in DataEntry
//   4. Unknown fields → diagnostics warning
```

## DataFlow

```
Environment init:
  schema.Register<string[]>("inherits")     // core
  schema.Register<int>("layer")              // core
  schema.Register<string>("Concept")         // GameConceptPlugin init
  schema.Register<string>("DefaultState")    // StateEnginePlugin init
  
Loader (JSON/CSV/etc):
  for each field in data:
    type = schema.Find(fieldName)
    if type is string[] → parse as array
    if type is string → parse as string
    if type is int → parse as number
    if type is object → parse as component (via PrimitiveRegistry)
    if unknown → warning + store as inferred type
  
  create DataEntry with typed fields (frozen)

DataGraphBuilder:
  layer sort → merge (replace entry for different layer, merge fields for same layer)
  ALL fields preserved automatically (no selective copying)
  no new DataEntry creation

DataCatalogBuilder:
  inheritance resolve → flat catalog
  fields preserved automatically

Plugin:
  entry.GetField<string>()    → "Enemy" (concept)
  entry.GetField<string[]>()  → ["BaseEntity"] (inherits)
  entry.GetField<int>()       → 0 (layer)
```
